## RésuméDe400mots
Dans ce post Twitter du 2 janvier 2026, Guillermo Rauch, CEO de Vercel, formule une thèse architecturale sur les agents de codage : leur abstraction fondamentale est le CLI (Command-Line Interface), non par préférence de facteur de forme, mais par nécessité technique d'accès à la couche OS.

Rauch redéfinit les agents de codage comme des "computer-use agents" plutôt que de simples assistants d'édition de code. Cette distinction est cruciale : ces agents exécutent des programmes, en créent de nouveaux, installent ceux qui manquent, scannent le système de fichiers et lisent les logs. Ils automatisent l'ordinateur à un niveau bas, bien au-delà de la manipulation de texte. Cette vision repositionne des outils comme Claude Code non comme une évolution du paradigme copilot (autocomplétion → assistance IDE), mais comme "IA pour votre système d'exploitation".

L'argument technique repose sur une analogie avec les éditeurs CLI comme nvim : ces outils fonctionnent aussi bien sur votre Mac que sur n'importe quelle machine accessible via ssh. Cette universalité devient critique dans un monde cloud où les environnements de développement sont distribués. Le CLI transcende les frontières machine locale/cloud, permettant aux agents de fonctionner indifféremment sur votre ordinateur de bureau ou sur des instances distantes.

Rauch projette ensuite une vision de scalabilité massive : lancer un million de Claude Codes concurrents pour s'attaquer à un bug, identifier des menaces cybersécurité, traiter le backlog d'issues, construire des features basées sur les retours utilisateurs, ou exécuter du QA. Cette vision de parallélisation massive nécessite une infrastructure capable d'absorber cette charge.

C'est ici qu'intervient Vercel Sandbox, présenté comme "la couche de calcul infini pour agents" et "la façon la plus sûre d'exécuter du code que vous n'avez pas écrit". Le post construit ainsi un narratif complet : de la thèse architecturale (CLI comme abstraction nécessaire) à la vision d'usage (millions d'agents concurrents) jusqu'à la solution infrastructure (Vercel Sandbox).

Le message clé est que les agents de codage ne sont pas une feature d'éditeur mais une nouvelle classe de logiciels d'automatisation système. Ils requièrent un accès OS profond et une infrastructure cloud élastique pour réaliser leur potentiel de parallélisation massive. Cette perspective transforme le débat sur les coding agents d'une discussion sur l'expérience développeur vers une question d'architecture systèmes et d'infrastructure cloud.

Rauch positionne ainsi Vercel non seulement comme plateforme de déploiement web mais comme infrastructure computing pour l'ère des agents autonomes.
