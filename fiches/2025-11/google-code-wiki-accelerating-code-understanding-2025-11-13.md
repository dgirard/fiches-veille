## RésuméDe400mots

Google lance Code Wiki en public preview, plateforme  adressant "one of biggest, most expensive bottlenecks in software development" : reading existing code. Appliquant mission Google ("organize world's information") aux développeurs, Code Wiki "unlocks vital knowledge currently buried in complex source code" via documentation continuously updated structured wiki.

**3 caractéristiques différenciatrices**

**Automated & always up-to-date** : Scanne full codebase, regenerates documentation after each change. "The docs evolve with the code" - contrairement static files stagnants. Automation élimine maintenance burden documentation.

**Intelligent & context-aware** : Entire always-current wiki serves knowledge base Gemini-powered integrated chat. Citation clé : "Not talking to generic model, but to one that knows your repo end-to-end." Chat agent comprend contexte complet repository, répond highly specific questions vs generic LLM responses.

**Integrated & actionable** : Every wiki section + chat answer hyper-linked directly relevant code files/definitions. "Reading and exploring merge into one workflow" - seamless navigation concept → implementation.

**Code Wiki Website : public preview aujourd'hui**

Ingests public repositories, generates/hosts/maintains comprehensive interactive documentation each one. Features :

**Navigation interactive** : Jump directly high-level concept explanations → exact code files/classes/functions referenced. Vs linear reading traditional docs.

**Gemini-powered chat** : Uses always-up-to-date wiki context answer repository-specific questions, "instantly bridging gap between learning about code and actually exploring it."

**Automatic diagrams** : Always-current architecture, class, sequence diagrams visualize complex relationships matching exact current state code. Diagrammes synchronisés code changes.

**Impact promis** : New contributors make first commit Day 1, senior developers understand new libraries minutes vs days.

**Gemini CLI Extension : waitlist ouvert**

While public repos addressées website, "often our own private repos hardest document effectively." Enterprise pain point : original code author might not available anymore, understanding legacy code massive hurdle. Code Wiki positioned "game-changer for internal environments."

Gemini CLI extension allows teams run same system locally and securely on internal repositories. Addressing enterprise security/privacy requirements tout en delivering same capabilities.

**Vision future déclarée**

"Developers should spend time building, not deciphering." "Era manual, outdated documentation and endless code-reading is over." "Future of development is about instant understanding."

Positioning audacieux : documentation traditionnelle obsolète, instant understanding nouveau standard.

**Dual go-to-market**

Public preview website (codewiki.google) immediate availability open-source community. Gemini CLI extension (waitlist) enterprise/private repos. Strategy covers spectrum use cases developer ecosystems.

Code Wiki represents Google Cloud Developer Experiences bet improving developer productivity via AI-powered automated documentation continuously synchronized codebase evolution, Gemini integration providing intelligent context-aware assistance.
