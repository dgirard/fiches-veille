## Pense-betes
- **Le problème des agents actuels** : Ils sont réactifs et asynchrones, mais demandent une supervision constante ("babysitting"). Cela crée une charge mentale pour le développeur qui doit "attendre et vérifier", brisant le flow.
- **Vision : Proactivité** : Des agents qui agissent comme des coéquipiers, anticipant les besoins et intervenant sans qu'on leur demande explicitement, en arrière-plan.
- **4 ingrédients de la proactivité** : Observation (contexte continu), Personnalisation (apprendre les habitudes), Timing (intervenir au bon moment), Ubiquité (s'intégrer dans le workflow existant).
- **Projet Jules (Google Labs)** : Un agent de codage proactif.
    - **Niveau 1 (Emergence)** : Détecte et corrige des petits problèmes (tests manquants, dépendances inutilisées) en arrière-plan. Le "Sous-chef".
    - **Niveau 2 (Contexte Projet)** : Comprend l'ensemble du projet, les frameworks, le style de l'équipe. Le "Manager de cuisine".
    - **Niveau 3 (Conscience Système)** : Collaboration multi-agents (Jules pour le code, Stitch pour le design, Insights pour la data) pour proposer des améliorations transversales basées sur la donnée réelle (ex: fixer une régression de performance qui impacte l'UX).
- **Fonctionnalités clés de Jules** : Mémoire éditable, Agent critique (adversarial), Vérification (écriture de scripts de test/screenshot), To-do bot (scanne le code pour les TODOs et propose de les faire).

## RésuméDe400mots
Kath Korevec, Directrice Produit chez Google Labs, présente une vision transformatrice pour les agents de développement : passer de la **réactivité à la proactivité**. Elle utilise l'analogie de la charge mentale domestique (devoir rappeler à quelqu'un de faire la vaisselle) pour décrire l'état actuel des agents IA. Même s'ils exécutent des tâches, ils obligent les développeurs à les surveiller, brisant leur concentration et leur "flow". Les humains sont des processeurs sériels, pas parallèles ; superviser 16 agents en arrière-plan n'est pas une expérience développeur viable.

La solution réside dans des agents proactifs capables de "faire la vaisselle sans qu'on leur demande". Korevec présente **Project Jules**, un agent autonome développé par Google Labs, conçu pour observer, apprendre et anticiper. Elle décrit trois niveaux d'évolution :
1.  **L'Assistant Attentif ("Sous-chef")** : Il nettoie derrière le développeur, ajoutant des tests manquants ou corrigeant des imports sans intervention explicite.
2.  **Le Gestionnaire de Projet** : Il comprend le contexte global, les frameworks utilisés et les préférences de l'équipe, agissant comme un partenaire qui connaît la "maison".
3.  **L'Intelligence Collective (Niveau visé)** : Jules collabore avec d'autres agents spécialisés (comme "Stitch" pour le design et "Insights" pour la data) pour comprendre les conséquences des changements. Par exemple, identifier qu'une baisse de conversion est due à une régression de performance et proposer un correctif global (code + design).

Techniquement, cela se traduit par des fonctionnalités comme la **mémoire persistante** (que l'utilisateur peut éditer), un **agent "critique"** qui challenge le code généré, et la capacité de **vérification autonome** (ex: écrire et exécuter un script Playwright pour prouver que le code marche). Korevec annonce des mises à jour pour décembre, incluant la détection proactive de "TODOs" dans le code pour les transformer en tâches exécutées automatiquement. L'objectif final est de libérer les développeurs des tâches de maintenance (friction) pour qu'ils se concentrent sur la création pure.
