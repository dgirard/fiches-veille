## Pense-betes

- Thèse centrale : le nombre de développeurs restera globalement stable malgré l'IA, à cause de l'effet Reine Rouge (compétition entre entreprises)
- Paradoxe de Jevons : les gains d'efficacité du vibe coding seront absorbés par une explosion du volume de code (30M → 1Md+ lignes)
- Analogie forte : sysadmins → DevOps/SRE. Les sysadmins n'ont pas disparu, ils ont changé de nom et de compétences (chaos engineering, déploiement continu, systèmes distribués)
- Les développeurs ne liront plus le code — ce n'est déjà plus soutenable. Il faut de nouvelles voies de compréhension
- Le rôle persiste, le titre change : « AI Wranglers », « Agentic Herders », « Human-AI system integrator »
- Confusion courante : penser que le rôle du développeur est d'écrire/lire du code = comme penser que le rôle du sysadmin est de câbler des serveurs
- Critique acerbe du court-termisme managérial : licencier des développeurs pour le cours de bourse, puis devoir les réembaucher plus cher
- « Alice » = métaphore du développeur licencié qui reviendra comme intégrateur humain-IA, plus cher
- Wardley préconise la reconversion (retraining) plutôt que le licenciement, mais prédit que les entreprises ne le feront pas
- L'ingénierie logicielle doit devenir une discipline d'ingénierie plutôt qu'un artisanat (craft → engineering)
- La compétition, les nouvelles industries et la componentisation sont les vrais moteurs de la demande

## RésuméDe400mots

Simon Wardley répond sous forme de dialogue socratique fictif à la question de savoir si les LLM et le vibe coding entraîneront plus ou moins de développeurs. Sa réponse est nuancée : probablement autant, en raison de l'effet Reine Rouge. Les entreprises étant en compétition, tout gain de productivité sera immédiatement réinvesti pour rester compétitif. Concrètement, une grande entreprise passera de 30 millions à plus d'un milliard de lignes de code, simplement pour maintenir sa position.

Wardley mobilise le paradoxe de Jevons comme cadre explicatif : lorsqu'une ressource devient plus efficace, sa consommation augmente plutôt que de diminuer. Le vibe coding rend la production de code moins coûteuse, ce qui provoquera une explosion du volume de logiciel produit, pas une réduction des effectifs.

L'analogie historique centrale est celle des sysadmins. Quand la virtualisation a rendu le racking de serveurs physiques obsolète, les sysadmins n'ont pas disparu. Ils se sont transformés en DevOps Engineers et SREs, acquérant de nouvelles compétences : chaos engineering, déploiement continu, systèmes distribués. De même, les développeurs ne disparaîtront pas mais évolueront vers des rôles de gestion de « meutes d'agents », de prise de décisions structurelles et de maintien d'une chaîne de compréhension sur des systèmes toujours plus complexes.

Wardley affirme que lire le code n'est déjà plus soutenable et que l'ingénierie logicielle doit se transformer d'artisanat en discipline d'ingénierie pour développer de meilleures méthodes de compréhension systémique. Le titre de « software engineer » disparaîtra probablement — non par disparition du rôle, mais parce que trop de dirigeants ont publiquement déclaré que ces profils ne seraient plus nécessaires et ne voudront pas perdre la face. De nouveaux titres émergeront : « Human-AI system integrator », « AI Wrangler », « Agentic Herder ».

La chute du dialogue est mordante. Wardley introduit « Alice », la développeuse licenciée sur les conseils d'un « thought leader ». Alice sera bientôt réembauchée, plus chère, sous un nouveau titre. Le court-termisme managérial — licencier pour faire monter les stock-options puis quitter le navire avant les conséquences — est pointé comme le vrai problème. Wardley recommande la reconversion plutôt que le licenciement, tout en prédisant cyniquement que les entreprises n'adopteront pas cette voie.
