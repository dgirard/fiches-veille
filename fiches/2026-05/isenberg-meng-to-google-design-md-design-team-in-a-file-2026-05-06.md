# isenberg-meng-to-google-design-md-design-team-in-a-file-2026-05-06

## Veille
Podcast Greg Isenberg × Meng To (designer, fondateur Design+Code, créateur des produits Aura / New Form / Dream Cut) sur **`design.md`** — la convention open-source de Google, équivalente à `agents.md` / `skills.md` / `soul.md` mais **pour la design system** (typographie, couleurs, spacing, WebGL/Three.js animations, règles de reveal). Idée centrale : porter "l'**âme du design**" dans un fichier markdown qui se transmet à un agent (Claude Code, Codex, OpenClaude, Gemini, Stitch, Aura, V0, Lovable, Cursor) pour préserver la **cohérence cross-medium** (web, mobile, slides Replit, motion design Hyperframes/Remotion). Triade enseignée : **HTML = plat fini, design.md = recette, skills = ingrédients** (skills typo, lasers, skeuomorphic, 3D — 63 dans New Form). Diagnostic majeur : **design drift** sur les workflows one-shot (`v0`, Lovable, Framer) qui démarrent forts puis dérivent en générique. Méta-message : la *taste* est le seul **moat** restant — *"si une chose ressemble à une autre, sa valeur baisse de 10× à 100×"*. Workflow : **Reference → Design.md → Generate → Inspect → Systemize → Iterate (jusqu'à 1000+ prompts) → Remix → Expand → Export**. Critique des **purple gradients** ("you just run") = baseline générique post-vibe-coding. Meng To revendique avoir dépensé ~500 000 $ en tokens, fait 1000–10 000 itérations par produit, gère 4 produits en parallèle en solo.

## Ton
**Profil** : Conversation podcast vidéo (~30-40 min), format Q&R, écran partagé pour démo live, registre informel-professionnel. Public cible : indie hackers, makers, vibe coders, fondateurs solo, designers en transition agentique. Niveau technique : intermédiaire — pas de code mais beaucoup de termes design (WebGL, Three.js, skeuomorphic, font smoothing, gradient, reveal animation, secondary button).

**Style** : Greg Isenberg pose des questions ouvertes courtes ("comment fais-tu pour...", "qu'est-ce que ça veut dire...") et laisse Meng dérouler. Meng a un style **demonstratif** — il partage l'écran, montre Aura, télécharge un design.md, l'attache au prompt, génère, change de plateforme (Stitch), montre les skills (63), passe en motion, en slide, en mobile. C'est de la **pédagogie par démo**, pas de la conférence. Phrases marquantes plantées pour viralité : *"baseline so high but also so generic"*, *"if you see purple gradients you just run"*, *"design drift"*, *"HTML = finished dish, MD file = recipe, skills = ingredients"*, *"taste is the real value here"*, *"AI is not making me lazier, AI is making me work more"*, *"if something looks like another thing, it reduces the value by 10× to 100×"*.

Position épistémique de Meng : **artisan-praticien** qui a fait 1000–10 000 itérations, qui dépense ses propres tokens (~500 000$), qui construit pour lui-même d'abord (*"only because we're building something for ourselves"*) puis ouvre. Ton émotionnel : reconnaissant envers Greg (*"I started at 3K MRR, shot up to 15K MRR through your podcast"*), engagé (*"I've never worked more in my entire life"*), un peu contestataire envers les big tech (*"unfair Google can give all this for free when I spend tokens"*).

Ton de l'épisode : **discours doctrinal naissant pour le design agentique** — Meng To fixe un vocabulaire (`design.md`, design drift, design memory, ingredients/recipe/dish, judgment per minute) qui n'est pas encore stabilisé dans le marché mais qui circule via le podcast. Greg amplifie en ramenant régulièrement la valeur business (*"better install rate, better buy rate, better share rate"*). C'est un **épisode-manifeste** plus qu'un tuto technique.

## Pense-betes

- **Date publication** : 2026-05-06 (chaîne YouTube Greg Isenberg). Vidéo : https://www.youtube.com/watch?v=oLu32YpiIJw
- **Identité du guest** : "Mang" / "Ming" / "Mangto" dans le transcript = **Meng To**, fondateur de **Design+Code** (designcode.io), designer reconnu depuis 10+ ans, autodidacte (drop out 18 ans, première société à 22 ans). Francophone d'origine — Greg note *"you come from the same town as me. You speak French"*. Multi-startup founder : Aura, New Form, Dream Cut (Mac app à venir), refonte de design code from scratch.
- **Quel `design.md` ?** : convention Google, **open-source**, sortie récente (annoncée en tweet avec une vidéo de 10min). Même logique que `agents.md` / `skills.md` / `soul.md` mais pour le design.
- **Triade conceptuelle** :
  - **HTML** = "finished dish" (plat fini)
  - **`design.md`** = "recipe" (recette : typographie, couleurs, spacing, fondations)
  - **Skills** = "ingredients" (skills typo, lasers, skeuomorphic, 3D…)
  - L'agent reçoit les trois et produit un résultat **cohérent à travers tous les médias** (web, mobile, slides, motion).
- **Problème nommé : design drift** — *"You start something, you're kind of satisfied, but then you get into the other stuff and it becomes completely different."* Symptôme : one-shot brillant suivi de pages génériques. Cause racine : pas de mémoire de design partagée entre prompts.
- **Solution : design memory transférable** — `design.md` voyage entre Lovable / Cursor / Codex / Claude Code / Stitch / Aura / V0 / Figma. Permet de *"commit it to memory"* en disant *"remember this"* à l'agent.
- **Workflow Meng To complet** :
  1. **Reference** (toujours commencer par copier — *"every designer who starts drawing always copies first"*)
  2. Charger **`design.md` + HTML** comme attachements de prompt
  3. **Generate** (premier shot)
  4. **Inspect**
  5. **Systemize** (figer le système)
  6. **Iterate** — *"this is where I get to a thousand iterations"* (Meng a fait jusqu'à **10 000 itérations** sur certains produits)
  7. **Remix** quand on veut changer de catégorie
  8. **Expand** vers d'autres médiums : slide, promo vidéo, Instagram, intros, Hyperframes, Remotion
  9. **Export** final
- **Iteration vs Remix** : iteration = on est satisfait, on incrémente petit (90% du temps Meng) ; remix = on veut une nouvelle catégorie de produit (10%).
- **Ressources gratuites** : Aura → section "design.md" avec design systems téléchargeables (MD + HTML). Variant.com (`variant.com`) pour la phase remix créative. Communautés Figma / Framer / Webflow donnent des **templates** mais Meng critique : *"closed behind techniques and rectangles and pixels of Figma, but how do you tell that to an agent?"* — d'où la valeur du `design.md` comme **blueprint exportable**.
- **Skills (New Form, 63 skills)** : copywriting (*"améliore ton copy 100%"*), lasers (*"everyone clicks on it, I don't know why"*), skeuomorphic (boutons "lickable" Steve Jobs), 3D (Three.js), batch design, font smoothing, system font, body font, secondary color/button. Chaque skill est **un prompt copyable**, gratuit, vendu via tokens uniquement.
- **WebGL / Three.js** : ce qui élève un design "from 0 to 50 or 50 to 80 real fast". Lasers, animations, 3D objets posés sur typographie. **Différenciant majeur** vs sites générique purple-gradient.
- **Critique du purple gradient** : 5-10 ans en arrière = wow ; aujourd'hui = signal "you just run". Métaphore plus large : *"baseline is so high but also generic"* — le défi est de **dépasser la baseline**, pas l'atteindre.
- **Mode compétitif redéfini** : *"as people starting new ideas, the only moat we have is we can click a button and say we found this new model that literally cooked all the designers"*. Survie par **recence d'adoption** des nouveautés (modèles, tools, conventions comme `design.md`).
- **Stack démo Meng** :
  - **Aura** (son startup) — design.md gallery, remix, prompts attachés
  - **Google Stitch (3.1)** — "infinite canvas", concurrent direct, *"unfair they give it for free"*
  - **OpenClaude / Codex / Claude Code** — agents locaux, lecture de fichiers `.md` en folders nestés, traitement multi-screens
  - **Variant.com** — phase remix créative
  - **Replit slides / Hyperframes / Remotion** — formats d'expansion (slides, motion design)
- **Économie indie revendiquée** : Meng To dépense **~500 000 $ en tokens** pour ses produits, fait 1000–10 000 itérations par produit, gère **4 produits en parallèle en solo** (Aura, New Form, Dream Cut, refonte Design+Code), équipe historique de Design+Code maintenue par fidélité humaine plus que nécessité opérationnelle.
- **Citation Steve Jobs (apocryphe assumée)** : *"if you want to go fast, go alone ; if you want to go far, go in a team."* Meng la mobilise pour justifier sa double posture solo-indie / team-fidélité.
- **"Judgment per minute"** : nouveau KPI du designer agentique — moins de pixel-pushing, plus de **micro-décisions par minute** sur ce que l'agent produit. *"AI is not intelligent enough to make smaller decisions yet, but eventually decisions will become bigger."*
- **Skills-maxi (citation indirecte)** : Meng se réfère à un précédent guest de Greg qui se définit "skills-maxi" — Meng confirme : *"I don't put anything in my agents.md because I have my design.md per project and my skills per workflow. If you only change agents.md, it costs more tokens AND it's too general."* **Spécialisation des fichiers MD par dimension** (project / workflow / design / agent).

- **Articulation avec la veille corpus** :
  - Convergence forte avec les fiches **agents.md / skills.md** (Vincent Superpowers, Forrestchang Karpathy Skills, Vercel Gao agents.md, Lattice atoms/molecules/refiners). `design.md` est l'**extension domain-specific de cette famille** au design system.
  - Lien direct avec **Cognitive Surrender / Comprehension Debt** (Osmani 2026-05-05) : Meng dit *"the first screen is not the hard part… you have no leverage to change it"* — c'est exactement le surrender d'Osmani transposé au design.

## RésuméDe400mots

Greg Isenberg reçoit le designer **Meng To** (fondateur Design+Code, créateur d'**Aura**, **New Form**, **Dream Cut**) pour décortiquer **`design.md`** — la convention open-source de Google qui porte "l'âme du design system" dans un fichier markdown, équivalente à `agents.md` / `skills.md` / `soul.md` mais dédiée à la couche visuelle (typographie, couleurs, spacing, WebGL / Three.js, règles de reveal animation). L'épisode est un **manifeste pédagogique** sur comment créer des designs *jaw-dropping* sans être designer.

Diagnostic central : le **design drift**. Les outils one-shot (V0, Lovable, Framer) génèrent un premier écran impressionnant puis dérivent en pages génériques sur les écrans suivants — pas de **mémoire de design** partagée entre prompts. La solution `design.md` agit comme **blueprint transférable** : il porte typographie, couleurs, spacing, WebGL et règles d'animation à travers Lovable, Cursor, Codex, Claude Code, Google Stitch, Aura, V0, Figma. Meng formalise une triade : **`design.md` = recette, HTML = plat fini, skills = ingrédients**. Les skills (63 dans New Form : lasers, skeuomorphic, 3D, copywriting, batch design) sont des prompts copyables qui dopent un design de "0 à 50 ou 50 à 80" en un clic.

Le workflow complet de Meng en 9 étapes : **Reference → design.md → Generate → Inspect → Systemize → Iterate (1000+ prompts) → Remix → Expand → Export**. Il revendique jusqu'à **10 000 itérations par produit**, **~500 000 $ dépensés en tokens**, **4 produits en parallèle en solo**. Distinction structurante : *iteration* (incrémental, 90% du temps) vs *remix* (changement de catégorie, 10%).

Démos enchaînées à l'écran : Aura (gallery design.md + remix + prompt), Google Stitch 3.1 (infinite canvas, "unfair they give it for free"), OpenClaude/Codex (lecture multi-fichiers MD en folders nestés), Variant.com (phase créative remix), Replit slides / Hyperframes / Remotion (expansion en slides + motion).

Méta-thèse : la **taste est le seul moat restant**. *"If something looks like another thing, it reduces the value by 10× to 100×."* Le purple gradient — wow il y a 5-10 ans — est devenu signal de fuite. La baseline est haute mais générique : la survie passe par la **recence d'adoption** des modèles/tools/conventions et par le **judgment per minute** (micro-décisions humaines sur ce que l'agent produit).

Greg conclut sur l'idée du **second brain design** : capturer l'inspiration croisée dans le réel et le digital pour la rappeler quand on crée. Meng valide : "ne laissez pas l'IA tout faire. Si vous le faites, assurez-vous que ce soit dans un workflow que vous maîtrisez déjà."
