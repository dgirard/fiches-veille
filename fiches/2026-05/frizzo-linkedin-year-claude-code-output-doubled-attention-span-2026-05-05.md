# frizzo-linkedin-year-claude-code-output-doubled-attention-span-2026-05-05

## Veille
Tribune LinkedIn d'Alexandre Frizzo après un an d'utilisation quotidienne de Claude Code, qui pose un **bilan nuancé** rare dans le corpus 2026 — productivité **multipliée par 3-5×** dans son cas (cohérent Wescale, et conforme à la médiane des praticiens committés ; le tail élite va beaucoup plus haut, cf. Cherny *few dozen PRs/day + 150 PRs record* et Karpathy *"peaks much higher than 10×"*), mais **coûts cognitifs cachés** assumés. Thèse-pivot : ***"the new bottleneck is supervision"*** — le métier a changé de forme, on ne *écrit* plus du code, on *décide* à propos du code généré par les agents. Gains : 3-5× output, projets précédemment infaisables désormais réalisables (yak-shaving, boilerplate), coût quasi nul de l'expérimentation. Pertes assumées : ***"writing muscle"*** atrophié (le code manuel semble *effortful*), **flow state profond rare** (context-switching constant entre supervisions), **satisfaction d'ownership diminuée** (*"code is good, but isn't quite mine"*). Tensions non résolues : **FOMO** (*"every hour I'm not at the keyboard is an hour an agent could be earning for me"*), **review quality** à 3-5× volume, **skill atrophy**. Statistiques mobilisées : médiane 3-4h coding effective sur 8h jour, **23 min** récupération contexte par interruption (étude Gloria Mark), 15-25 min entrée en flow, 500% productivité en flow (McKinsey). Position épistémique exemplaire : refuse simultanément le narratif *"AI is bad"* et l'enthousiasme acritique. Contre-poids salutaire à Cherny *"coding is solved"* (2026-05).

### Le métier a changé de forme

- **Avant** : *"write hard parts, review easy parts"* (le développeur écrit la logique difficile et review les détails).
- **Après** : *"deciding about code"* sur de **multiples codebases simultanément** — le développeur est devenu un superviseur d'agents.
- **Le bottleneck a changé** :
  - **Ancien bottleneck** : protéger le *deep work state* (entrée en flow, concentration profonde).
  - **Nouveau bottleneck** : la **supervision à grande vitesse** (reading agent output, deciding correctness, integrating, catching subtleties).
  - *"The traditional focus on protecting deep work state became irrelevant once code generation became fast."*
  - *"Quality control at scale now requires entirely different defenses."*

### Gains assumés

- **3-5× productivité** sur une journée typique. Ce chiffre est **cohérent avec Wescale X3-X4** (2026-05-03) et reflète l'expérience d'un praticien committé sur 1 an. À ne pas confondre avec le tail élite, qui va beaucoup plus haut : **Cherny** (2026-05) déclare *"a few dozen PRs/day, 150 PRs in a single day record"* (clairement >10× sur la base habituelle), **Karpathy** (2026-04-29) précise *"10× is not the speed up — people who are very good at this peak a lot more than 10×"*, **Curran/Intercom** (2026-04-16) mesure que le **top 5% des performers génère 6× le median PR throughput**. Lecture juste : **distribution inégale**, 3-5× pour la médiane committée, 10×+ pour le tail élite. Le mythe à dégonfler n'est pas le 10× lui-même mais *"10× automatique pour tous avec un simple prompt"*.
- **Projets précédemment infaisables désormais réalisables** : yak-shaving, boilerplate, automatisations marginales — la *long-tail* devient adressable (cohérent **Wescale 2026-05-03** *"besoins historiques restés trop coûteux peuvent enfin être adressés"*).
- **Coût quasi nul de l'expérimentation** — barrière mentale levée pour les essais.

### Pertes assumées

- **"Writing muscle"** atrophié — *"manual coding feels effortful now"*. Métaphore corporelle qui rend tangible la **dégradation par non-usage**.
- **Deep flow state rare** — *"constant context-switching"* entre supervisions multiples. Le flow état cognitif protégé par 30 ans de littérature *Deep Work* (Cal Newport) devient inaccessible.
- **Ownership satisfaction diminuée** — ***"the code is good, but isn't quite mine."*** Le sens psychologique de la création est érodé.

### Tensions non résolues (les questions qu'il ne tranche pas)

1. **FOMO Problem** — savoir que Claude peut travailler la nuit crée une **pression psychologique**. *"Every hour I'm not at the keyboard is an hour an agent could be earning for me."* Le travail weekend/soir devient tentant **même quand les projets sont terminés**. Inversion de la logique du *deep work* : ce n'est plus l'absence de productivité qui pèse, c'est l'absence de **supervision active**.
2. **Review Quality at 3-5× Volume** — reviewer à 3-5× le volume risque le *skimming*. Les pratiques qualité historiques de l'industrie supposaient un **rythme humain**. À ce niveau de débit, les *subtle bugs* peuvent passer la review.
3. **Skill Atrophy** — Frizzo pose la **question ouverte** : *"does the writing muscle still matter, or is it becoming commoditized?"* Il ne tranche pas — c'est un point politique.

### Position éthique-épistémique

- **Refus simultané des deux narratifs disponibles** :
  - *"AI is bad"* — Frizzo le rejette explicitement.
  - *Uncritical enthusiasm* — il le rejette aussi.
- **Posture tierce** : gains réels + coûts réels, **tensions non résolues** plutôt que conclusions.
